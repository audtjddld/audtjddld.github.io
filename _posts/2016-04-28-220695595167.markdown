```
@Override
 @SuppressWarnings("unchecked")
 protected void doReadPage() {

  EntityTransaction tx = null;

  if (transacted) {
   tx = entityManager.getTransaction();
   tx.begin();

   entityManager.flush();
   entityManager.clear();
  }//end if

  Query query = createQuery().setFirstResult(getPage() * getPageSize()).setMaxResults(getPageSize());

  if (parameterValues != null) {
   for (Map.Entry me : parameterValues.entrySet()) {
    query.setParameter(me.getKey(), me.getValue());
   }
  }

  if (results == null) {
   results = new CopyOnWriteArrayList();
  }
  else {
   results.clear();
  }

  if (!transacted) {
   List queryResult = query.getResultList();
   for (T entity : queryResult) {
    entityManager.detach(entity);
    results.add(entity);
   }//end if
  } else {
   results.addAll(query.getResultList());
   tx.commit();
  }//end if
 }
```

노란색으로 칠한 곳에서 문제가 발생되더라구요.

디버깅으로 @StepScope를 제거한 상태에서 동작하게되면 EntityManager에 값이 들어 있습니다.

그런데 @StepScope를 사용하면 EntityManager값이 null로 되어있기 때문에 발생되는 문제인데요.

해결 방법을 찾다가 아래 링크를 보게되었고

<http://stackoverflow.com/questions/30330879/could-not-autowired-object-in-itemstreamreader-open-method> <http://stackoverflow.com/questions/30330879/could-not-autowired-object-in-itemstreamreader-open-method>

IteamReader 쪽 method에서 reader.open(new ExecutionContext());

을 하게되니 정상적으로 동작.

하지만 이유를 모르겠다 -_-;;

stackoverflow 작성자도 자기는 spring 초보라서 이유를 설명해 달라고 적혀 있을 뿐..
